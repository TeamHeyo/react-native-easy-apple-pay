# Contributing

## Setup

```bash
npm install        # also runs `bob build` via the `prepare` hook
npm run typecheck
npm run lint
npm test
```

## Layout

- `src/` — TypeScript source
- `ios/` — Swift + ObjC bridge code
- `__tests__/` — Jest tests (use the global `__RNEasyApplePayStub` to intercept native calls)
- `__mocks__/react-native-easy-apple-pay.ts` — ships with the package so consumers can mock the library in their own tests
- `example/` — runnable Expo dev-client app, links the library via `file:..`
- `lib/` — generated by `bob build`; do **not** commit

## Test loop

```bash
npx jest --watch
```

## Running the example app

See `example/README.md`. Requires a Mac with Xcode, a developer account, and a real device with a card in Wallet.

## New Architecture

To verify TurboModule path:

```bash
cd example
RCT_NEW_ARCH_ENABLED=1 npm run ios:device
```

## Releasing

1. Update `version` in `package.json` (the podspec reads from it).
2. Update `CHANGELOG.md`.
3. `npm run prepack` runs typecheck, lint, tests, and build.
4. `npm pack --dry-run` to inspect tarball contents.
5. `npm publish --provenance` (CI workflow draft at `.github/workflows/release.yml` once an `NPM_TOKEN` secret is configured).

## Style

- ESLint + Prettier are wired up; CI fails on lint errors.
- Strict TypeScript (`noImplicitAny`, `noUnusedLocals`, `noUnusedParameters`).
- No `console.log` in source — use `console.warn` for genuinely surprising states only.
- iOS code: dispatch UIKit work to `DispatchQueue.main`; null out captured completion handlers after invocation.
